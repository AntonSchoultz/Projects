Overview

Use Cases
Use cases are implemented in POJO classes, usualy as void methods with no parameters.
These methods are marked with a MetaTask annotation which indicates icons, tool tips etc.
The class is then 'registered' with the TaskManager.
Use cases can then be executed via the TaskManager using an 'action-key' of the form SimpleClassName@methodName
Menues and buttons can be created and added to a display simply by refering to the action-key.
TaskManager can enable/disable tasks (use-cases).
Buttons will be enabled/disabled in line with the TaskManager enable/disable.
The actions can be executed in-line or as separate threads / delayed / AWT thread etc.
Icons
GuiIconManager provides the functionality to retrieve Icons by name, but also allows for compositing/overlaying of icons.
The Icon set typically comprises of main icons (eg Document, Script ...)
and 'modifier' icons (+ for add, - for remove, etc) which can be overlayed.
The MetaTask annotation can refer to an icon with a text expression.
eg: 'document+remove' will create a new icon starting with document.png and then overlaying any non-transparent areas from remove.png.
The resulting icon is then cached against it's reference 'document+remove'.
User application can also get an icon from the manager by specifying a key/text-expression.
Data Access
Data objects are also implemented as POJO classes. MetaFieldInfo annotations are use to provide additional information about the data fields.
Default screens for editing these objects (or fields) can be auto-generated.
GuiConfigScreen can create an edit screen for any simple data object.
xml
Provides simplified XML support to parse, create and query xml structures.
util
Various utilities, including functionality to persist and reload data objects as xml documents. (makes use of xml package)
